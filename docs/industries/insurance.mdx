# Insurance — claims & fraud rings

**The problem.** Most claims are honest and independent. Organised fraud isn't — it
hides in the *connections* between claims that look unrelated one at a time: the same
phone number on two policies, the same repair shop across a dozen claims, the same
address shared by claimants who supposedly don't know each other. A flat claims table
can't see a ring. A graph can: model claims, policies and the parties they share, and
a single traversal lights up the cluster.

This scenario centres on **graph entity resolution** — turning a suspicious claim into
the ring it belongs to.

## 1. Model claims and the parties they share

Claims, policies and parties (people, phone numbers, addresses, repair shops) are all
resources; the edges record who is connected to what. The shared parties are what make
a ring visible.

## 2. Expand a suspicious claim into its ring

Walk out from the flagged claim. The returned sub-graph is everything connected to it
— and any *other* claim that shares a party, phone or shop with it is part of the same
cluster.

<Tabs groupId="lang">
<TabItem value="java" label="Java">

```java
ResourceNetwork ring = client.resources().fetchRelated("claim_88421", 4);

// other claims reachable through shared parties/phones/shops
ring.nodes().stream()
    .map(Resource::getExternalId)
    .filter(id -> id.startsWith("claim_") && !id.equals("claim_88421"))
    .forEach(c -> System.out.println("linked claim: " + c));
```

</TabItem>
<TabItem value="python" label="Python">

```python
ring = client.resources.fetch_related(external_id="claim_88421", depth=4)

linked = [n.external_id for n in ring.nodes
          if n.external_id.startswith("claim_") and n.external_id != "claim_88421"]
print("linked claims:", linked)
```

</TabItem>
<TabItem value="rust" label="Rust">

```rust
use dataplatform_rust_sdk::resources::RelatedResourcesForm;

let ring = api.resources.fetch_related(
    &RelatedResourcesForm::from_external_id("claim_88421").with_depth(4)).await?;

for node in ring.nodes() {
    if node.external_id.starts_with("claim_") && node.external_id != "claim_88421" {
        println!("linked claim: {}", node.external_id);
    }
}
```

</TabItem>
</Tabs>

The more claims a single party, phone or shop connects, the stronger the signal —
exactly the shared-node reasoning behind
[alarm correlation](/guides/correlate-alarms), applied to entities instead of assets.

## 3. Flag and watch the trend

Raise a `fraud_review` [event](/guides/detect-events) on the cluster, and track
`claims_per_shop` or `claims_per_phone` as [series](/guides/query-and-aggregate) — a
node whose claim count climbs unusually fast is a ring forming in real time.
