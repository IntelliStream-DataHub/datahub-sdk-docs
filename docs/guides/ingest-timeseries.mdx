# Live monitoring: high-throughput ingestion

A common job: push a steady, high-volume stream of sensor readings into DataHub so a
dashboard or alerting rule can act on them. The SDK parallelises ingestion for you — you
hand it the data, it **chunks into batches and sends them concurrently**.

:::note How it works
Datapoints are chunked (default **10,000** per request — the analytics store is optimised
for large batches), fanned out across virtual threads bounded by a concurrency limit, and
transient failures are retried. You get back an `IngestResult` summarising what landed and
what didn't.
:::

## Ingest a million readings

<Tabs groupId="lang">
<TabItem value="java" label="Java">

```java
var client = DatahubClient.fromEnv();

// readings: Map<String, List<Datapoint>> grouped by time-series external id
IngestResult result = client.timeseries().ingest(readings,
    IngestOptions.builder()
        .batchSize(10_000)     // datapoints per request
        .parallelism(16)       // concurrent in-flight batches
        .maxRetries(3)
        .build());

System.out.printf("ingested %,d, failed %,d%n", result.succeeded(), result.failed());
```

</TabItem>
<TabItem value="python" label="Python">

```python
client = DataHubClient.from_env()

result = client.timeseries.ingest(
    readings,                  # dict[str, list[tuple[datetime, float]]]
    batch_size=10_000,
    parallelism=16,
    max_retries=3,
)
print(f"ingested {result.succeeded:,}, failed {result.failed:,}")
```

</TabItem>
<TabItem value="rust" label="Rust">

```rust
let api = datahub_sdk::create_api_service();

let result = api.time_series
    .ingest(readings, IngestOptions { batch_size: 10_000, parallelism: 16, max_retries: 3 })
    .await?;

println!("ingested {}, failed {}", result.succeeded, result.failed);
```

</TabItem>
</Tabs>

## Then query an aggregate

Roll the raw readings up to hourly averages for a chart:

<Tabs groupId="lang">
<TabItem value="java" label="Java">

```java
var hourly = client.timeseries().aggregate("engine_temperature",
    Aggregate.AVG, Granularity.HOUR, last(Duration.ofDays(1)));
```

</TabItem>
<TabItem value="python" label="Python">

```python
hourly = client.timeseries.aggregate(
    "engine_temperature", agg="avg", granularity="1h", last_days=1)
```

</TabItem>
<TabItem value="rust" label="Rust">

```rust
let hourly = api.time_series
    .aggregate("engine_temperature", Aggregate::Avg, Granularity::Hour, last_days(1))
    .await?;
```

</TabItem>
</Tabs>

:::warning Ordering
Batches are sent in parallel, so there is **no cross-batch ordering guarantee** — which is
fine for time-stamped data, since each datapoint carries its own timestamp.
:::
